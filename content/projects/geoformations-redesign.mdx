## Problem / Context

Educational and research organizations needed websites that explained complex topics clearly, supported different audience types, and remained editable by non-technical teams.

## Approach

I treated each redesign as both a UX and requirements problem.

- Mapped user journeys before visual design.
- Clarified page goals and conversion intent per section.
- Used lightweight usability checks to validate decisions before launch.

## Execution

- Built wireframes and prototypes in Figma.
- Implemented production builds in WordPress.
- Added analytics and behavior tools to monitor drop-offs and navigation friction.
- Iterated content hierarchy and CTA order based on observed usage.

## Results

- Engagement improved by 40% on key project surfaces.
- Users reached high-intent content faster with fewer navigation loops.
- Stakeholders gained a reusable pattern for future content additions.

## Learnings

Clear information architecture does more for trust and usability than visual polish alone.
